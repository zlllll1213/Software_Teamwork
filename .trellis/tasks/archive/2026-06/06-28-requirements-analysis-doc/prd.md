# 需求说明书分析文档

## Goal

基于新增的《知识管理系统 — 需求说明书》和《智能问答系统 — 需求说明书》，结合用户补充的产品决策，沉淀一份完整的功能需求分析 Markdown 文档，作为后续架构设计、任务拆分和实现评估的依据。

## What I already know

- 用户要求完整需求分析，不需要 MVP 缩减版。
- 两份新增 PDF 已提取文本，并且仓库中已有对应的原始需求说明文档：
  - `docs/knowledge_management_system.md`
  - `docs/smart_quote_system.md`
- 系统包含知识管理和智能问答两个强关联模块。
- 用户补充的关键决策：
  - 会话历史持久化使用服务端数据库。
  - 知识库页面对所有登录用户可见；普通用户只允许查看 `public` 知识库。
  - 向量数据库使用 Qdrant。
  - 原文下载需要权限校验和审计日志。
  - 未来会提供问答准确率考核标准，包括标准问题集、标准答案和评分规则。
  - 需要支持图片 OCR。
  - 模型能力需要支持多供应商。
  - 报告生成先按后续模块处理。

## Requirements

- 在 `docs/需求分析/知识管理智能问答智能分析.md` 中维护完整需求分析文档。
- 文档需要覆盖：
  - 系统目标和边界
  - 用户角色与权限
  - 知识库管理
  - 文档管理
  - 文档解析、OCR、切片、向量化处理管线
  - Qdrant 检索与权限过滤
  - 智能问答、意图识别、多轮会话和 SSE
  - 引用溯源、原文下载、审计日志
  - 多供应商模型配置
  - 统计监控
  - 问答准确率评测预留
  - 核心数据对象
  - 待确认问题与风险
- 文档应明确“完整范围”，不以 MVP 方式收缩。

## Acceptance Criteria

- [x] Markdown 文档已落盘。
- [x] 文档内容能独立说明完整功能需求。
- [x] 文档纳入用户已确认的 9 条补充决策。
- [x] 已结合 `docs/knowledge_management_system.md`、`docs/smart_quote_system.md`、`docs/system.md` 和 `docs/report_generation_system.md` 补充内容。
- [x] 不改动业务代码。

## Definition of Done

- 需求分析文档已完成。
- 运行基础检查确认文件存在且 Markdown 内容可读。
- 汇报新增文件路径和未处理事项。

## Out of Scope

- 不实现功能代码。
- 不创建数据库 schema、API 或前端页面。
- 不把报告生成纳入本轮详细需求范围，只标记为后续模块。

## Technical Notes

- 当前任务只涉及文档持久化。
- PDF 原文仍为未跟踪文件，不在本任务中处理归档或提交策略。
