# 智能问答系统需求说明书

## 1. 概述

本模块旨在建设一个面向电力行业的智能问答系统，基于大语言模型（LLM）和检索增强生成（RAG）技术，为用户提供知识检索、智能对话和答案引用溯源能力，帮助用户快速从知识库中获取准确信息。

> 注意：最终项目的问答准确率将作为考核指标。

---

## 2. 用户登录与身份认证

- 系统应支持用户注册和登录功能，用户通过用户名和密码进行身份认证。
- 系统应对所有 API 请求进行身份认证，未登录用户无法访问系统功能。
- 用户登录后应维持会话状态，页面刷新后无需重复登录。

---

## 3. 智能对话

- 系统应提供基于大语言模型的智能对话能力，支持用户以自然语言进行交互。
- 对话应支持流式输出（SSE），实时展示生成内容。
- 系统应支持多轮对话管理，维护对话上下文。
- 用户侧应支持多会话管理，支持创建新对话、切换历史对话、删除对话。
- 会话历史应支持本地持久化存储，页面刷新后自动恢复。

---

## 4. 意图识别与路由

- 系统应支持自动识别用户意图，并根据意图路由到对应处理模块。
- 支持的意图应包括但不限于：
  - 知识问答
  - 一般对话

---

## 5. 知识增强检索

- 系统应支持基于 RAG（Retrieval-Augmented Generation，检索增强生成）的知识问答。
- 系统应将用户问题与知识库中的相关文本片段匹配后生成回答。
- 检索应支持跨多个知识库的联合检索。
- 检索应支持以下模式：
  - 语义向量检索
  - 向量检索 + 重排序（Rerank）
- 检索应支持：
  - 相似度阈值过滤
  - 重排序分数阈值过滤
- 系统应可配置检索返回的 Top K 结果数。

---

## 6. 引用溯源（加分项）

- 系统生成的回答应支持引用标注，标注回答中引用到的知识库文档片段。
- 用户应可查看每个引用对应的：
  - 原文片段
  - 来源文档名称
  - 相关性分数
- 引用标注应支持多个连续引用来自同一文档时的合并显示。
- 系统应支持通过引用的文档 ID 追溯并下载原文文件。

---

## 7. 思考过程展示

- 系统在生成回答时应支持展示多步思考过程，让用户了解处理链路并获得及时反馈。
- 思考步骤应支持折叠和展开。
- 默认在流式输出完成后自动折叠。

---

## 8. 管理功能

### 8.1 问答配置管理

系统应提供知识问答配置管理页面，支持管理员运行时配置问答模块参数。

配置参数至少包括：

- 术语知识库选择
- 技术监督知识库选择
- Top K
- 相似度阈值
- 重排序阈值

### 8.2 检索体验测试

系统应提供检索体验测试页面，支持管理员验证知识库检索效果。

### 8.3 大语言模型配置

系统应提供大语言模型配置页面，支持管理员运行时修改以下参数：

- 模型 API 地址
- API 密钥
- 模型名称
- 超时时间

默认云端 API 参考：

https://cloud.siliconflow.cn/me/models

### 8.4 权限控制

管理功能仅限以下角色访问：

- 管理员（Admin）
- 超级管理员（Super Admin）

---

## 9. 数据统计与监控

### 9.1 管理后台概览

系统应提供管理后台概览页面，展示智能问答相关的核心指标。

### 9.2 统计指标

统计指标至少包括：

- 知识问答总次数

### 9.3 趋势分析

系统应提供近 30 天知识问答趋势图表。
