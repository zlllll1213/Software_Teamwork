# 知识管理系统 — 需求说明书

## 1. 概述

本模块旨在建设一个面向电力行业的知识管理系统，支持知识库的全生命周期管理、文档的上传与解析处理，为知识问答和报告生成提供基础数据支撑。

---

## 2. 用户登录与身份认证

- 系统应支持用户注册和登录功能，用户通过用户名和密码进行身份认证。
- 系统应对所有 API 请求进行身份认证，未登录用户无法访问系统功能。
- 用户登录后应维持会话状态，页面刷新后无需重复登录。

---

## 3. 知识库管理

- 系统应支持创建、编辑、删除知识库，每个知识库具有独立的名称和描述。
- 知识库应支持按文档类型进行分类，包括但不限于：
  - 规程规范
  - 技术报告论文
  - 术语条目
  - 通用文档
- 知识库应支持配置检索策略，至少支持：
  - 语义向量检索
  - 向量检索 + 重排序
- 知识库应支持展示以下元信息：
  - 文档数量
  - 创建时间
  - 创建人
- 知识库列表应支持：
  - 按类型过滤
  - 按关键词搜索
- 知识库应支持批量删除操作。

---

## 4. 文档管理

- 系统应支持向知识库中上传文档，支持格式包括但不限于：
  - PDF
  - DOCX
  - PPTX
  - XLSX
  - MD
  - TXT
  - 常见图片格式
- 上传文档时支持为文档添加标签，用于后续检索过滤。
- 系统应支持：
  - 拖拽上传
  - 点击选择文件上传
- 系统应支持文档处理全流程状态跟踪，包括：
  - 已上传
  - 解析中
  - 切片中
  - 向量化中
  - 就绪
  - 失败
- 处理失败的文档应支持查看错误原因并重试处理。
- 系统应支持删除文档：
  - 单个删除
  - 批量删除
- 就绪状态的文档应支持查看切片详情。
- 文档列表应支持：
  - 分页浏览
  - 按状态筛选
- 文档应支持编辑标签信息。
- 当知识库的分段策略或检索策略变更时，系统应支持自动触发后台重新处理所有就绪状态的文档。

---

## 5. 文档解析与处理

- 系统应支持将上传的文档解析为文本化内容。
- 系统应根据知识库的分段策略对解析后的内容进行语义切片。
- 系统应支持将切片内容进行向量化嵌入，并持久化存储至向量数据库。
- 文档处理管线应支持并发控制，可配置最大并发解析任务数。
- 系统应支持多种文档解析后端，允许运行时切换。

---

## 6. 前台知识检索

- 系统应提供面向普通用户的知识检索页面，支持用户在前台直接搜索知识库中的文档内容。
- 检索应支持跨多个知识库的联合搜索，用户可自由选择检索范围。
- 检索应支持：
  - 语义向量检索
  - 向量检索 + 重排序
- 检索应支持相似度阈值过滤，确保返回结果的相关性。
- 检索应支持基于文档标签的元数据过滤，帮助用户精确定位。
- 检索结果应展示：
  - 文档名称
  - 相关度分数
  - 匹配内容摘要
- 系统应支持配置检索返回的 Top K 结果数。
- 检索页面应与知识库管理联动，用户仅可检索其有权限访问的知识库。

---

## 7. 系统配置

- 系统应提供嵌入模型的运行时配置能力，包括：
  - 模型名称
  - API 地址
  - 向量维度

  默认云端 API 参考：

  https://cloud.siliconflow.cn/me/models

- 系统应提供重排序模型的运行时配置能力，包括：
  - 模型名称
  - API 地址
  - Top N

  默认云端 API 参考：

  https://cloud.siliconflow.cn/me/models

- 以上配置变更应支持运行时生效，无需重启服务。

---

## 8. 模型本地部署（加分项）

- 系统应支持嵌入模型在内网环境下本地化部署，实现向量化处理的完全离线运行。
- 系统应支持重排序模型在内网环境下本地化部署，实现检索重排序的完全离线运行。
- 以上模型本地部署后，系统应保持与使用云端 API 一致的功能体验。

---

## 9. 数据统计与监控

- 系统应提供管理后台概览页面，展示知识管理相关的核心指标。
- 统计指标应至少包括：
  - 知识库数量
  - 文档数量
  - 切片数量
- 系统应提供近 30 天文档上传趋势图表。
