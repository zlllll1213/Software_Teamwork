## 6. 缺陷与处理记录

| 问题 | 等级 | 处理结论 | 关联 issue / PR | 说明 |
| --- | --- | --- | --- | --- |
| ... | 小问题 / 大问题 | 已修复 / 已转 issue / 暂不处理 | `#xxx` | 复现步骤、影响范围、验证命令和风险 |

判断规则：

- 小问题：测试主责人可以在本任务 PR 中顺手修复，但必须说明修复范围、验证命令和风险。
- 大问题：不要在测试任务里扩大修改范围；应新建独立 issue 指派给对应 owner 小组，并在本报告中链接。

大问题包括但不限于：跨服务契约变更、数据模型或 migration 变更、权限或安全边界缺陷、需要 owner service 重构、需要产品或架构决策、会影响多个模块的行为变更。
