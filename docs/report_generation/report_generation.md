# 报告生成 MCP 工具与后端接口需求文档

## 1. 文档说明

本文是报告生成功能的后端需求文档，结合了两部分信息：

- 最新明确边界：不做用户认证，不做前端页面，不做模型服务配置；只提供后端接口和 MCP 工具能力。
- 原始 PDF 业务细节：支持迎峰度夏检查报告和煤库存审计报告两类固定专业报告，覆盖大纲生成、逐章节内容生成、Word 导出、报告记录、模板管理、素材管理和数据统计。

本文只描述后端能力需求，不展开前端页面设计、登录注册、权限系统、模型配置页面、数据库建表 SQL 或部署方案。

相关文档：

- [报告生成后端接口文档](api_interfaces.md)
- [报告生成数据模型文档](data_models.md)

## 2. 系统定位

### 2.1 可行性结论

可以做。

本功能应定位为大项目中的一个后端报告生成服务模块。它不拆成多个内部微服务，但对外提供两类能力：

- 后端业务接口：通过 gateway 的 `/api/v1` 公开资源型 HTTP API，供大项目前端、管理后台、其他后端模块或工具调用方使用。
- MCP 工具接口：供其他小组、智能体或自动化流程调用；工具执行过程中如需访问报告生成 HTTP 能力，也必须调用 gateway `/api/v1` 公共接口，不直接调用 `document` 服务内部地址。

公开 HTTP API 必须遵循主仓库最新接口规范：先更新 `docs/api/gateway.openapi.yaml`，再作为稳定前端契约使用；路径使用 RESTful 资源名，不在 path 中出现 `generate`、`regenerate`、`export`、`retry`、`download` 等动作词。报告生成公开接口已升级为 gateway OpenAPI active paths，owner service 为 `document`。

### 2.2 不做的内容

本模块明确不负责：

- 用户注册。
- 用户登录。
- 用户认证。
- 角色权限校验。
- 前端页面。
- 管理后台页面。
- 模型 API 地址、密钥、模型名称、超时时间等模型服务配置。

如果调用方需要鉴权、菜单、角色或页面展示，应由大项目其他模块负责。本模块只提供经 gateway 暴露的后端接口能力。

### 2.3 本模块负责的内容

本模块负责报告生成相关后端能力：

- 报告创建接口。
- 报告类型和模板查询接口。
- 大纲生成接口。
- AI 重新生成接口。
- 大纲编辑接口。
- 正文生成接口。
- 正文和表格编辑接口。
- 报告保存接口。
- DOCX 导出接口。
- 报告记录管理接口。
- 模板管理接口。
- 素材管理接口。
- 数据统计接口。
- 管理类接口。
- MCP 工具接口。
- 数据库存储和通过 file 服务进行的文件存储。

## 3. 调用方说明

| 调用方 | 使用方式 |
|---|---|
| 大项目业务前端 | 通过 gateway 调用报告创建、编辑、导出、记录查询等公开接口 |
| 大项目管理端 | 通过 gateway 调用模板、素材、统计数据和生成任务等管理类公开接口 |
| 其他后端模块 | 通过 gateway 调用报告生成公开接口生成报告或获取报告记录 |
| MCP 调用方 | 通过 MCP 工具生成大纲、生成正文、查询任务状态、查询模板结构、触发导出；工具内部 HTTP 调用仍走 gateway |
| 项目评审人员 | 通过大项目已有展示方式查看最终报告效果 |

本模块可以接收 gateway 注入或 MCP 工具上下文传递的 `operator_id`、`operator_name`、`operator_type` 等字段用于记录创建人和操作来源，但不负责校验身份合法性。

## 4. 范围说明

### 4.1 范围内

- 两类固定报告类型：迎峰度夏检查报告、煤库存审计报告。
- 报告主题和基础信息录入接口。
- 根据报告类型、模板和要求生成大纲。
- 基于最新报告信息、模板和要求重新生成大纲、正文或指定章节。
- 多级大纲查询、编辑、删除、排序和自动重新编号。
- 根据大纲逐章节生成报告正文。
- 生成任务状态和进度查询。
- 章节正文和表格内容编辑接口。
- 报告结构化数据保存到数据库。
- 模板文件、素材文件、导出文件通过 file 服务保存到底层对象存储。
- DOCX Word 文档导出。
- 已保存报告重新导出 Word，不重新执行 AI 生成。
- 报告记录分页、查看、下载、删除。
- 模板上传、查看、编辑、删除和结构配置。
- 素材上传、查看、删除和引用。
- 数据统计接口。
- 管理类后端接口。
- MCP 工具接口。
- 生成任务状态、失败原因、操作日志和追溯信息。

### 4.2 范围外

- 用户认证与登录。
- 角色权限体系。
- 前端页面。
- 管理后台页面。
- 模型服务配置。
- 大项目全局组织架构。
- 将报告功能拆成多个独立微服务。
- 完整多人协同编辑。
- 完整审核流。
- 报告版本对比和版本回滚。
- 复杂素材检索和知识库问答，除非后续阶段明确纳入。

## 5. 总体业务流程

### 5.1 后端接口调用流程

1. 调用方通过 gateway 选择报告类型：迎峰度夏检查报告或煤库存审计报告。
2. 调用方通过 gateway 查询或选择报告模板。
3. 调用方通过 gateway 提交报告主题和基础信息。
4. 本模块创建报告草稿并保存到数据库。
5. 调用方创建报告生成任务资源，请求生成大纲。
6. 本模块根据报告类型、模板、主题和上下文生成多级大纲，并产出新的大纲资源版本。
7. 调用方通过接口编辑大纲，包括修改标题、删除章节、调整顺序和重新编号。
8. 调用方可创建新的报告生成任务资源，请求 AI 重新生成完整大纲；重新生成应基于最新报告信息、模板和要求。
9. 调用方创建正文生成任务资源。
10. 本模块根据当前大纲逐章节生成正文内容。
11. 调用方可创建新的任务或章节版本，请求 AI 重新生成完整正文或指定章节正文。
12. 调用方查询生成任务进度和章节生成结果。
13. 调用方通过接口编辑章节正文和表格内容。
14. 本模块保存最终报告数据。
15. 调用方请求导出 DOCX Word 文档。
16. 本模块生成 DOCX 文件并通过 file 服务保存底层文件对象。
17. 调用方获取报告文件 ID、`contentPath` 或通过文件内容接口读取导出文件。
18. 本模块记录报告、生成任务、导出文件和操作日志。

### 5.2 MCP 调用流程

1. MCP 调用方调用报告生成工具。
2. 调用方传入报告类型、模板、报告主题、上下文、生成要求和素材引用。
3. 本模块校验业务参数是否完整。
4. 本模块创建生成任务并生成唯一请求标识。
5. 本模块按调用目标生成大纲或正文。
6. 本模块返回结构化结果、任务状态、警告信息和错误信息。
7. 若生成耗时较长，调用方可通过状态查询工具获取进度。
8. 若调用方请求保存结果，MCP 工具应通过 gateway 公开接口进入报告生成流程；本模块将报告数据写入数据库，将相关文件通过 file 服务保存。

## 6. 功能需求

### FR-001 报告类型查询

系统应提供报告类型查询接口。

需求明细：

- 支持查询迎峰度夏检查报告。
- 支持查询煤库存审计报告。
- 每种报告类型应返回类型编码、类型名称、描述、可用模板信息。
- 报告生成前必须指定报告类型。

验收标准：

- 调用方可以查询两种固定报告类型。
- 不同报告类型可以绑定不同模板。

### FR-002 报告创建与基础信息接口

系统应提供报告创建和基础信息保存接口。

需求明细：

- 支持填写报告名称。
- 支持填写报告主题。
- 支持指定报告类型。
- 支持指定报告模板。
- 支持填写专业。
- 支持填写电厂或业务对象。
- 支持填写年份。
- 支持填写补充说明。
- 支持保存草稿。
- 支持记录 gateway 注入或 MCP 工具上下文传递的创建人或来源标识。

验收标准：

- 调用方可以创建报告草稿。
- 草稿可通过报告记录接口查询。
- 草稿重新读取后已填写内容不丢失。

### FR-003 报告大纲生成

系统应支持根据报告主题、报告类型、模板和上下文生成报告大纲。

需求明细：

- 支持后端接口生成大纲。
- 支持 MCP 工具生成大纲。
- 大纲应支持多级章节结构，包括章、节、子节等。
- 大纲应基于预定义模板生成固定结构。
- 大纲应与报告类型匹配。
- 大纲生成请求应记录任务状态。
- 大纲生成失败时应记录失败原因。
- 大纲生成失败后应允许重试。
- 支持基于最新报告基础信息、模板结构和生成要求重新生成完整大纲。
- 重新生成大纲时应创建新的生成任务记录，不应覆盖报告基础信息。

验收标准：

- 输入报告主题和必要上下文后可以生成大纲。
- 大纲具有多级结构。
- 大纲符合模板约束。
- 生成失败时不丢失已录入报告信息。
- 调用方可以重新生成大纲，并查询本次重新生成任务状态。

### FR-004 报告大纲编辑接口

系统应提供大纲编辑接口。

需求明细：

- 支持修改章节标题。
- 支持新增章节。
- 支持删除章节。
- 支持调整章节顺序。
- 支持自动重新编号。
- 支持保存编辑后的大纲。
- 支持将编辑后的大纲作为正文生成依据。
- 支持大纲编辑后由调用方按指定章节触发 AI 重新生成对应正文。

验收标准：

- 删除章节后编号自动更新。
- 调整顺序后编号自动更新。
- 修改标题后正文生成使用新标题。
- 编辑后重新读取报告，大纲不丢失。

### FR-005 报告正文生成

系统应支持根据大纲逐章节生成报告正文内容。

需求明细：

- 支持后端接口生成正文。
- 支持 MCP 工具生成正文。
- 支持逐章节生成。
- 支持记录每个章节的生成进度。
- 支持记录总体进度，例如已完成章节数 / 总章节数。
- 支持正文内容生成。
- 支持表格数据处理和呈现。
- 支持章节生成后编辑。
- 支持基于最新大纲、上下文、模板和已编辑内容重新生成完整正文。
- 支持基于指定章节重新生成单章节正文。
- 支持部分章节失败时保留已生成内容。
- 支持失败后重试。

验收标准：

- 调用方可以基于当前大纲生成正文。
- 调用方可以查询总体进度和章节进度。
- 每个章节生成后可以查询内容。
- 表格内容可以结构化返回。
- 失败后已生成内容不丢失。
- 调用方可以重新生成完整正文或指定章节正文。

### FR-006 报告内容编辑接口

系统应提供报告内容编辑接口，用于支持上层系统实现在线编辑。

需求明细：

- 支持编辑章节正文。
- 支持编辑表格内容。
- 支持保存编辑结果。
- 支持读取最新编辑结果。
- 支持编辑后由调用方请求 AI 重新生成指定章节。
- 支持导出时使用最终编辑结果。

验收标准：

- 编辑内容保存后不丢失。
- 重新读取报告时返回最新编辑内容。
- 导出的 Word 文档包含最终编辑内容。

### FR-007 报告保存与存储

系统应支持将报告数据保存到数据库，并将文件类对象通过 file 服务保存到底层对象存储。

需求明细：

- 报告基础信息保存到数据库。
- 报告大纲保存到数据库。
- 报告章节正文保存到数据库。
- 生成任务记录保存到数据库。
- 操作日志保存到数据库。
- 模板文件通过 file 服务保存。
- 素材文件通过 file 服务保存。
- 导出 DOCX 文件通过 file 服务保存。
- 数据库中应保存 file 服务文件引用。
- 文件保存失败时应记录错误。
- 数据库保存失败时应返回错误。

验收标准：

- 已保存报告可以重新读取。
- 导出文件可以通过 file 服务读取。
- 文件存储异常时接口返回明确错误。

### FR-008 Word 文档导出

系统应支持将生成后的报告导出为 DOCX Word 文档。

需求明细：

- 支持导出 DOCX 文件。
- 导出内容应保留最终编辑结果。
- 导出文档应遵循统一样式。
- 样式至少包括字体、字号、标题层级、段落间距、表格样式、页眉页脚。
- 支持基于已保存报告数据重新生成 Word 文档。
- 重新导出时不需要重新执行 AI 生成。
- 支持返回导出文件标识和 `contentPath`；不得向公开调用方返回 MinIO 对象引用、file 内部 ID、内部 URL 或完整下载签名。

验收标准：

- 调用方可以成功触发 DOCX 导出。
- DOCX 文件可以用 Word 打开。
- DOCX 内容与最终保存内容一致。
- 同一报告重新导出时不重新生成正文。

### FR-009 报告记录管理接口

系统应提供报告记录管理接口。

需求明细：

- 记录报告名称。
- 记录报告类型。
- 记录专业。
- 记录电厂或业务对象。
- 记录年份。
- 记录生成时间。
- 记录状态。
- 支持分页查询报告记录。
- 支持按名称、类型、年份、状态、创建时间筛选。
- 支持查看报告详情。
- 支持基于保存数据重新生成 Word。
- 支持重新获取已生成文件引用。
- 支持删除不再需要的报告记录。

验收标准：

- 生成报告后记录列表出现对应记录。
- 记录中包含关键元数据。
- 历史报告可以重新获取。
- 删除记录后列表不再展示该记录。

### FR-010 模板管理接口

系统应提供报告模板管理接口。

需求明细：

- 支持上传报告模板文件。
- 支持查询模板列表。
- 支持查询模板详情。
- 支持编辑模板元数据。
- 支持删除模板。
- 模板可以绑定报告类型。
- 模板可以设置启用或停用。
- 模板结构可以通过接口配置。
- MCP 工具可查询可用模板结构。

验收标准：

- 模板可以上传并通过 file 服务保存。
- 模板上传后可在列表中查询。
- 模板可以被报告选择使用。
- 删除或停用模板后不可被新报告使用。

### FR-011 模板结构配置接口

系统应提供模板结构配置接口。

需求明细：

- 支持配置章节结构。
- 支持配置模板与报告类型的绑定关系。
- 支持配置导出样式作为加分项。
- 支持保存模板结构配置。
- 支持查询模板结构配置。

验收标准：

- 调用方可以保存模板结构。
- 调用方可以读取模板结构。
- 修改后的模板结构能影响后续大纲或导出结果。

### FR-012 素材管理接口

系统应提供专业素材管理接口。

需求明细：

- 支持上传专业素材文档。
- 支持查询素材列表。
- 支持查询素材详情。
- 支持删除素材。
- 支持素材与报告类型或模板关联。
- 素材可作为报告生成时的参考资料。

验收标准：

- 素材可以上传并通过 file 服务保存。
- 上传后的素材可以查询。
- 素材可以被生成任务引用。

### FR-013 数据统计接口

系统应提供报告生成相关统计接口。

需求明细：

- 返回报告生成相关核心指标。
- 至少返回报告模板数量。
- 至少返回报告生成记录数量。
- 返回近 30 天报告生成趋势数据。
- 返回生成成功、失败、进行中数量。
- 返回导出次数作为加分项。

验收标准：

- 调用方可以获取统计数据。
- 统计数量与实际数据一致。
- 趋势数据能反映近 30 天报告生成变化。

### FR-014 管理类接口

系统应提供管理类后端接口。

需求明细：

- 支持管理报告模板。
- 支持管理专业素材。
- 支持配置模板结构。
- 支持查看报告记录。
- 支持查看生成任务。
- 支持查看异常日志。
- 支持获取统计概览。

说明：

- 本模块只提供接口，不提供管理页面。
- 本模块不做管理员身份校验；权限控制由 gateway 或上层系统完成，本模块只消费已传递的调用上下文并记录审计信息。
- 本模块不提供模型服务配置接口。

验收标准：

- 调用方可以通过管理类接口维护模板和素材。
- 调用方可以查询生成任务和异常日志。
- 管理类操作有日志记录。

### FR-015 MCP 工具能力

系统应提供 MCP 工具能力，使其他小组或智能体可以结构化调用报告生成能力。

建议 MCP 工具包括：

- `generate_report_outline`：生成报告大纲。
- `regenerate_report_outline`：基于已有报告重新生成报告大纲。
- `generate_report_text`：生成报告正文。
- `regenerate_report_text`：基于已有报告重新生成完整正文。
- `regenerate_report_section`：重新生成指定章节正文。
- `get_generation_status`：查询生成任务状态。
- `get_report_result`：查询报告生成结果。
- `export_report_docx`：触发报告导出。
- `get_template_schema`：查询模板结构。

需求明细：

- 每个 MCP 工具应有明确名称、用途说明、输入参数和输出结构。
- MCP 工具输入参数应声明必填字段和可选字段。
- MCP 工具应返回结构化错误信息。
- MCP 工具调用应记录请求来源、调用时间、参数摘要、执行状态和错误信息。
- MCP 工具不应暴露密钥、内部存储路径或敏感配置。
- 删除、覆盖、批量重建等高风险操作不应默认通过 MCP 工具开放。

验收标准：

- 外部调用方可以通过 MCP 工具生成大纲。
- 外部调用方可以通过 MCP 工具生成正文。
- 外部调用方可以查询生成状态。
- MCP 调用失败时返回明确错误码和错误说明。

### FR-016 生成任务管理

系统应记录并管理生成任务。

需求明细：

- 每次生成大纲或正文应创建生成任务。
- 生成任务应有唯一请求标识。
- 生成任务应记录任务类型。
- 生成任务应记录发起人或调用来源。
- 生成任务应记录状态。
- 生成任务应记录开始时间和结束时间。
- 生成任务应记录失败原因。
- 生成任务应支持状态查询。

任务状态至少包括：

- `pending`
- `running`
- `succeeded`
- `partial_succeeded`
- `failed`
- `canceled`

验收标准：

- 调用方或 MCP 工具可以查询任务状态。
- 管理类接口可以查询任务列表和失败原因。

### FR-017 操作日志与审计

系统应记录关键操作日志。

需求明细：

- 记录报告创建。
- 记录大纲生成。
- 记录正文生成。
- 记录内容编辑保存。
- 记录模板变更。
- 记录素材变更。
- 记录 DOCX 导出。
- 记录管理类操作。
- 记录 MCP 工具调用。

验收标准：

- 调用方可以通过管理类接口查看关键操作记录。
- 发生问题时可以通过日志定位操作和调用来源。

## 10. 非功能需求

### 10.1 可用性

- 长时间生成任务应支持状态查询。
- 生成失败时应保留已输入内容。
- 导出失败时应允许重新导出。
- 接口错误应返回明确错误码和错误说明。

### 10.2 性能

- 报告列表应分页。
- 模板列表应分页。
- 素材列表应分页。
- 大文档生成应支持异步任务或流式生成。
- DOCX 导出应支持后台处理。
- 统计查询应避免每次全量实时计算。

### 10.3 安全边界

- 本模块不做用户登录、会话创建和权限系统。
- 公开 HTTP API 由 gateway 完成 bearer token 认证，并向 `document` 服务传递用户上下文；前端、管理端、其他后端模块和 MCP 工具的 HTTP 请求都不得绕过 gateway 直连 `document`。
- `document` 服务应使用 gateway 注入的 `X-User-Id`、`X-User-Roles`、`X-User-Permissions` 和 `X-Request-Id` 做审计、追踪和必要的服务边界校验。
- MCP 工具不应暴露系统密钥。
- MCP 工具不应返回 MinIO 内部敏感地址、file 内部 ID 或内部 URL。
- 日志中应避免记录敏感信息。

### 10.4 存储可靠性

- 数据库保存失败时应返回明确错误。
- file 服务保存失败时应返回明确错误。
- 数据库记录与 file 服务文件引用应保持一致。
- 对象不存在、文件损坏、下载失败等情况应有错误处理。

### 10.5 可追溯性

- 每次 MCP 调用应记录。
- 每次生成任务应记录。
- 每次导出应记录。
- 每次模板变更应记录。
- 每次管理类操作应记录。

## 11. 推荐阶段范围

### 11.1 第一阶段必须实现

- 两种固定报告类型。
- 报告创建接口。
- 模板选择接口。
- 创建报告生成任务资源，用于大纲生成。
- 创建报告生成任务资源，用于大纲重新生成。
- 大纲编辑接口。
- 创建报告生成任务资源，用于正文逐章节生成。
- 创建报告生成任务资源，用于正文重新生成。
- 创建章节版本或生成任务资源，用于指定章节重新生成。
- 生成进度查询接口。
- 正文和表格编辑接口。
- 报告保存到数据库。
- 文件通过 file 服务保存。
- 创建报告文件资源和读取文件内容。
- 报告记录列表接口。
- 模板上传和管理接口。
- 素材上传和管理接口。
- 管理类查询接口。
- MCP 生成大纲。
- MCP 生成正文。
- 生成任务状态记录。
- 基础统计接口。

### 11.2 第一阶段可简化

- 模板结构配置先支持 JSON 结构，不做复杂可视化能力。
- 素材管理先支持上传、列表和引用，不做复杂检索。
- 统计先支持模板数量、报告数量和近 30 天趋势。
- MCP 工具先开放生成和查询能力，不开放删除类能力。
- DOCX 样式先实现统一默认样式，后续再做复杂配置。

### 11.3 后续增强

- 更细粒度的重新生成策略，例如只重新生成表格、图片说明或指定子节。
- 素材解析和检索增强。
- 报告版本管理。
- 多人协同编辑的后端支撑。
- 审核流程的后端支撑。
- 生成质量评分。

## 13. 待确认问题

1. MCP 工具是否必须独立部署，还是作为报告生成服务的一部分暴露？无论哪种部署方式，对外 HTTP 访问边界都应保持为 gateway `/api/v1`。
2. 两类报告的标准模板文件是否已经存在？
3. 报告主题之外还必须填写哪些结构化字段？
4. “电厂”字段是否固定，还是应抽象为业务对象？
5. 素材是否必须参与生成，还是只作为附件和人工参考？
6. 表格数据来源是人工录入、素材解析、数据库查询，还是生成能力自动生成？
7. DOCX 样式第一阶段是否只要求统一默认样式？
8. 删除报告记录采用软删除还是物理删除？
9. 导出文件下载链接是否需要有效期？
10. 是否需要支持 PDF 导出？
11. AI 重新生成是否需要保留历史版本或差异对比？

## 15. 结论

本功能应建设为大项目中的单一后端报告生成服务模块，而不是拆成多个内部微服务。

本模块不做用户认证，不做前端页面，不做模型服务配置。它的核心交付是后端接口和 MCP 工具，支撑大项目完成“报告创建 - 大纲生成 - 大纲编辑 - 正文逐章节生成 - 正文编辑 - 数据库存储 - file 服务文件存储 - DOCX 导出 - 报告记录 - 模板管理 - 素材管理 - 统计查询 - MCP 工具调用”的完整闭环。
